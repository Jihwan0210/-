## 1️⃣ 프레임워크(Framework)에 대한 설명으로 옳은 것은?

### 문제
1. 소프트웨어 구성에 필요한 기본 구조를 제공함으로써 재사용이 가능하게 해준다.
2. 소프트웨어 개발 시 구조가 잡혀 있기 때문에 확장이 불가능하다.
3. 소프트웨어 아키텍처(Architecture)와 동일한 개념이다.
4. 모듈화(Modularity)가 불가능하다.

### ✅ 정답
**1번**

### 📌 핵심 정리
- 프레임워크는 소프트웨어 개발을 위한 **기본 구조(뼈대)** 제공
- 공통 기능 재사용 → **생산성·유지보수성 향상**
- **제어의 역전(IoC, Inversion of Control)** 개념 적용

---

## 2️⃣ HDLC 프레임 구조 순서

### ✅ 정답
**플래그 → 주소부 → 제어부 → 정보부 → FCS → 플래그**  
(**Flag → Address → Control → Information → FCS → Flag**)

### 📌 핵심 정리
- **Flag**: 프레임 시작·종료 표시 (`01111110`)
- **Address**: 송수신 스테이션 주소
- **Control**: 프레임 종류·순서·흐름 제어
- **Information**: 실제 데이터 (선택적)
- **FCS**: 오류 검출 (CRC)

## 3️⃣ 소프트웨어 형상관리에 관한 설명으로 거리가 먼 것은?

### 문제
1. 소프트웨어에서 일어나는 수정이나 변경을 알아내고 제어하는 것을 의미한다.
2. 소프트웨어 개발의 전체 비용을 줄이고, 개발 과정의 방해 요인을 최소화하는 것을 목적으로 한다.
3. 형상관리를 위하여 구성된 팀을 **chief programmer team**이라고 한다.
4. 형상관리에서 중요한 기술 중 하나는 **버전 제어 기술**이다.

### ✅ 정답
**3번**

### 📌 핵심 정리
- 형상관리는 **구성관리(Configuration Management)** 라고도 한다.
- 소프트웨어 변경 사항을 **체계적으로 관리·추적·통제**
- 작업 산출물을 **형상 항목(Configuration Item)** 으로 관리
- 버전 관리, 변경 관리가 핵심 기술
- **Chief Programmer Team**은 중앙 집중형 프로젝트 조직 개념으로 형상관리와 직접적인 관련 없음

## 4️⃣ 라우터(Router)에 관한 설명으로 틀린 것은?

### 문제
1. 네트워크 계층을 지원한다.
2. 전송되는 패킷들의 경로를 결정한다.
3. 게이트웨이(Gateway) 기능을 지원한다.
4. 브릿지(Bridge) 기능만을 지원한다.

### ✅ 정답
**4번**

### 📌 핵심 정리
- 라우터는 **네트워크 계층(Layer 3)** 장비
- 패킷의 **최단 경로 결정**
- 서로 다른 네트워크 연결 → **게이트웨이 기능 수행**
- 브릿지는 **데이터 링크 계층(Layer 2)** 장비

## 6️⃣ 소프트웨어 생명주기 모델 중 V-모델과 관련한 설명으로 틀린 것은?

### 문제
1. 요구 분석 및 설계 단계를 거치지 않으며 항상 통합 테스트를 중심으로 V 형태를 이룬다.  
2. Perry에 의해 제안되었으며 세부적인 테스트 과정으로 구성되어 신뢰도 높은 시스템을 개발하는 데 효과적이다.  
3. 개발 작업과 검증 작업 사이의 관계를 명확히 드러내 놓은 폭포수 모델의 변형이라고 볼 수 있다.  
4. 폭포수 모델이 산출물 중심이라면 V 모델은 작업과 결과의 검증에 초점을 둔다.  

### ✅ 정답
**1번**

### 📌 핵심 정리
- V 모델은 **요구 분석 및 설계 단계를 필수적으로 거친다**
- 폭포수 모델의 **변형**
- 요구사항 분석 → 설계 → 코딩 → 테스트 단계로 구성
- 각 단계는 **순차적으로 진행**
- 생명주기 **초기부터 테스트 작업을 지원**
- 요구사항·설계 단계에서 테스트 기준을 미리 정의

## 7️⃣ 다음 국제표준 통신 프로토콜 중 IP 주소를 물리 주소로 변환하기 위해 사용되는 것은?

### 문제
1. ARP  
2. TCP  
3. ICMP  
4. DHCP  

### ✅ 정답
**1번**

### 📌 핵심 정리
- **ARP(Address Resolution Protocol)**  
  - IP 주소 → **MAC 주소** 변환
- **RARP(Reverse ARP)**  
  - MAC 주소 → IP 주소 변환
- TCP: 전송 계층 프로토콜
- ICMP: 오류 및 제어 메시지 전달
- DHCP: IP 주소 자동 할당

## 9️⃣ 다음 설명에 해당하는 디자인 패턴은?

> 상속을 사용하지 않고 객체 간의 결합을 통해 유연하게 기능을 확장하는 구조 패턴

### 문제
1. Singleton 패턴  
2. Observer 패턴  
3. Decorator 패턴  
4. Bridge 패턴  

### ✅ 정답
**3번**

### 📌 핵심 정리 (Decorator Pattern)
- 기존 객체를 감싸 **동적으로 기능 확장**
- 상속 없이 기능 확장 가능

#### 구성 요소
- **Component**: 기본 인터페이스
- **ConcreteComponent**: 기본 기능 구현
- **Decorator**: Component를 감싸는 추상 클래스
- **ConcreteDecorator**: 추가 기능 실제 구현

#### 사용 예
- GUI 위젯
- 네트워크 스트림
- 입출력 처리

### ❌ 오답 피하기
- **Singleton**: 인스턴스 1개만 생성
- **Observer**: 상태 변화 통지
- **Bridge**: 구현과 추상 분리

- 직사각형은 **처리(Process)** 가 아님 → 오답 포인트

## 1️⃣2️⃣ 기억장치 배치 전략(First Fit) 문제

### 문제
기억공간이 **15K, 23K, 22K, 21K** 순으로 빈 공간이 있을 때  
배치 전략으로 **First Fit**을 사용하여 **17K** 프로그램을 적재할 경우  
**내부 단편화의 크기**는?

1. 5K  
2. 6K  
3. 7K  
4. 8K  

### ✅ 정답
**2번 (6K)**

### 📌 핵심 정리
- First Fit: **처음으로 적재 가능한 공간에 할당**
- 15K → 불가  
- 23K → 적재 가능 → 선택
- 내부 단편화 = 23K − 17K = **6K**

#### 🔹 내부 단편화
- 할당된 메모리 공간 − 실제 사용 공간

#### 🔹 배치 전략 정리
- **First Fit**: 처음 맞는 공간
- **Best Fit**: 가장 알맞은 최소 공간
- **Worst Fit**: 가장 큰 공간 선택

## 2️⃣0️⃣ OSI 7계층 중 종점 간 데이터 전송을 담당하는 계층은?

### 문제
1. 응용 계층  
2. 전송 계층  
3. 표현 계층  
4. 물리 계층  

### ✅ 정답
**2번**

### 📌 핵심 정리 (OSI 7계층)

- **물리 계층**: 전기적·물리적 전송
- **데이터 링크 계층**: 오류 제어, 흐름 제어
- **네트워크 계층**: 경로 설정, 논리 주소
- **전송 계층**: 종단 간 연결, 오류 제어, 흐름 제어
- **세션 계층**: 연결 설정·유지·종료
- **표현 계층**: 암호화, 압축, 코드 변환
- **응용 계층**: 사용자 서비스 제공
